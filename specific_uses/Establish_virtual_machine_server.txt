
1.出发点？
嵌入式开发人员的工作往往需要在公司的服务器上完成，而一般来说，出于保密的目的，服务器的环境外网是不能访问的。
因此，下班离开公司后想自己搞点东西(提升技能)的话，就需要一个 Linux 系统的环境(主要还是出于编译的目的)。
我有尝试过在个人笔记本上安装 ubuntu & win10 双系统，结论就是 ubuntu 的桌面系统体验比较差， 屏幕的色温难以控制， 时间长了眼睛会很难受；默认不能登陆 root 账号，修改需要花些时间，而登陆普通账号则需要经常去 sudo 获取 root 权限； 软件兼容性不好，毕竟是免费软件，维护肯定比不上 Windows, 需要花时间去自行修改一些设置； 
图形UI体验比较差， 但我说不出差在哪，就是不如 Windows 来的舒服，网上是有将 ubuntu 桌面美化成 MacOS 风格的教程， 但说实话，可以做但没有必要，因为我们需要的主要是内核和编译的资源，有个命令行就够了。
目前我觉得，最舒服的工作方式还是在 Windows 上工作，同时访问 Linux 服务器环境(这种软件有很多)， 可以很方便地进行分窗，可以同时打开多个登陆窗口，节约窗口切换的时间。而且 Windows 上有丰富的集成开发环境。
因此，决定通过虚拟机搭建一个用于学习测试的服务器。
